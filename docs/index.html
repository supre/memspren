Overview

Core concept: Every entity is a node. Every connection is a link. Your thinking partner maintains persistent context without manual effort.

Memspren is a multi-platform integration for managing your second brain across different platforms and tools. It implements a Zettelkasten (slip-box) system that keeps your knowledge graph atomic, linked, and queryable.

Plugins

Claude Skill

A Claude skill that turns Claude into your thinking partner for daily check-ins, capturing projects, ideas, people, and tasks as linked atomic notes in your Obsidian vault.

  • No CLI required — uses Claude's native file tools
  • Daily check-in workflow — brain dump → targeted follow-up → entity creation
  • Persistent memory — maintains context across sessions
  • Full Zettelkasten — YAML frontmatter + Obsidian wikilinks

Setup

  1. In Claude Code (Cowork mode), select your Obsidian vault folder
  2. Load the managing-second-brain.skill file from the claude/ folder
  3. Start a conversation — Claude will guide you through setup on first use

OpenClaw Skill

An OpenClaw skill that integrates memspren with OpenClaw—a local AI assistant that runs on your machine with browser control, file access, and extensible skills.

  • Local-first AI — runs on your machine, keeps your data private
  • Obsidian integration — same Zettelkasten patterns as Claude plugin
  • Daily check-in workflow — brain dump → targeted follow-up → entity creation
  • Full Zettelkasten — YAML frontmatter + Obsidian wikilinks

Setup

  1. In OpenClaw, load the memspren.skill file from the openclaw/ folder
  2. Point it to your Obsidian vault on first use
  3. OpenClaw will guide you through setup (~2 minutes)

Architecture

Two-Location Design

Every plugin follows this pattern:

Location What lives here
Skill folder config.md, Memory/, operational files
Vault All note content: Work/, People/, Log/, etc.

This separation keeps your knowledge graph clean while giving the AI system predictable places to store state.

Entity Types

  • Projects — deadline-driven work with goals and progress notes
  • Ideas — atomic thoughts, underdeveloped concepts, seeds for future projects
  • People — everyone you work with, interaction logs, connections to projects
  • Tasks — unified inbox with priority and connections to projects/ideas
  • Learnings — things you've learned, read, understood, connected to active work
  • Daily logs — one note per check-in, hub of all entities touched that day

Vault Structure

[vault_path]/
├── Vision/          — values, long-term vision, guiding principles
├── Strategy/        — quarterly goals and active strategies
├── Work/
│   ├── Projects/    — deadline-driven work
│   └── Ideas/       — lightweight captures
├── People/          — work contacts and interaction logs
├── Tasks/           — unified task inbox
├── Log/Daily/       — one note per check-in day
├── Notes/Learnings/ — things learned, connected to work
├── Inbox/           — unsorted catch-all
└── Archive/         — nothing ever deleted

Getting Started

  1. Pick a plugin that matches your workflow
    • Claude Skill if you use Claude Code (Cowork mode)
    • OpenClaw Skill if you prefer a local AI assistant
  2. Follow its setup instructions (takes ~2 minutes)
  3. Start capturing:
    • Projects you're working on
    • Ideas and thoughts
    • People you interact with
    • Tasks and reminders
    • Learnings and reflections

Philosophy

This isn't just a storage system. The goal is persistent context without manual effort:

  • Your AI reads recent notes on every interaction
  • You never have to re-explain yourself
  • Your thinking is captured atomically, linked, and queryable
  • Nothing gets deleted — everything gets archived
  • Your vault is portable, future-proof, and yours forever

Future Plugins
